Ark post-fix audit

This pass addresses the data-loss and wrong-session failures first. It does not claim the product or visual design is finished.

Branch: newark-ssh-tmux | Scope: current working tree | Date: 2026-07-09

Resolved in this pass

Durable history no longer has a 120-message cap

Capture reconciliation preserves the complete stored event list and updates a growing assistant response instead of replacing history with a reduced terminal snapshot.

Session and message writes are serialized

Store and per-session message mutations now use small in-process queues, preventing overlapping poll/send writes from clobbering each other.

Restart and Resume now recreate tmux

The endpoint closes the terminal stream, kills the old tmux session, starts a new one, restores logging/context, and uses codex ... resume --last for Resume.

Old captures cannot replace the active session

The browser tracks capture requests per session and ignores responses after the user switches away.

Unsafe model/reasoning fallbacks were removed

Ark sends a numeric control choice only when that choice was parsed from the live menu.

Failed sends restore the draft

Text, attachments, and pending state are restored when the request fails.

Parser deletion rules were narrowed

Normal replies mentioning project files, GPT models, warnings, tips, or sentences beginning with Model/Directory are no longer rejected wholesale.

Polling payload is bounded

Normal capture reads 2,000 tmux lines; one-time adopted-session import still requests the available 100,000-line history.

Corrupt readable storage stops writes

Malformed session, message, profile, settings, or secret files now surface an error instead of silently becoming empty data.

Profile routing rejects signed-out and full profiles

Auto selection now skips profiles that need login or reached max_concurrent.

Known remaining work

No user authentication

Source IP remains the only boundary. Keep Ark on a fully trusted LAN/tailnet until an app token or login is added.

Device identities are not merged

SSH aliases and Tailscale peers can still represent the same physical machine twice.

Assistant persistence is still parser-backed

Safer reconciliation avoids destructive rewrites, but structured tool transcripts are still preferable to terminal heuristics.

Resume uses --last, not an exact thread ID

Ark still needs to capture and persist each tool's exact conversation identifier.

CSS remains overgrown

The multiple chat polish layers and stale selectors should be consolidated after the repaired interaction flow is visually tested.

Feature diagnostics remain self-reported

Several GUI smoke assertions still check hard-coded capability flags rather than behavior.

Remote execution is prompt-mediated

Central Codex is told to use SSH; Ark does not yet provide a transparent remote filesystem/command adapter.
